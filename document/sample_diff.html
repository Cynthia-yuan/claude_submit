版本差异文档

文档版本： V1.0
对比版本： V1.0 → V2.0
生成日期： 2026-03-30
说明： 本文档记录了V1.0版本升级到V2.0版本的所有变更内容

第1章 概述

1.1 文档目的

本文档用于记录系统升级过程中的所有变更，帮助用户了解版本差异。

1.2 适用范围

适用于从V1.0升级到V2.0的所有用户。

第2章 系统架构变更

2.1 架构概述

V2.0版本采用了微服务架构，替代了原有的单体架构设计。

2.2 组件变更

组件名称 变更类型 说明
API网关 新增 统一API入口
旧认证模块 删除 被OAuth2.0替代
数据库连接池 修改 升级到HikariCP

第3章 功能变更

3.1 新增功能

  • 多语言支持（中/英/日）
  • 数据导出功能（支持Excel、PDF、CSV格式）
  • 实时消息通知

3.2 删除功能

  • 旧版报表系统 - 已被新BI系统替代
  • 邮件提醒功能 - 迁移至消息中心
  • 本地文件导入 - 安全风险，已下线

3.3 修改功能

  • 用户登录界面UI优化，支持扫码登录
  • 搜索功能增加模糊匹配和历史记录

第4章 配置变更

4.1 配置文件变更

配置文件从 config.properties 更改为 config.yaml

4.2 环境变量

变量名 旧值 新值
DB_HOST localhost db-cluster.internal
LOG_LEVEL INFO DEBUG
MAX_THREADS 100 200

4.3 端口变更

  • Web服务端口：8080 → 8081
  • 管理端口：9090 → 9091

第5章 交付件差异

注意： 本章详细列出了两个版本之间的文件和组件差异，请仔细核对。

5.1 交付件清单

5.1.1 安装包变更

文件名 变更类型 路径 说明
perf.tar.gz 删除 /opt/packages/perf.tar.gz 性能监控工具包，已内置
agent-v2.0.jar 新增 /opt/packages/agent-v2.0.jar 新版监控代理
setup.sh 修改 /opt/setup.sh 安装脚本更新
legacy-tools.zip 删除 /opt/packages/legacy-tools.zip 旧版工具集，不再维护
dashboard-v2.war 新增 /opt/packages/dashboard-v2.war 新版仪表盘

5.1.2 配置文件变更

文件名 变更类型 路径
application.properties 删除 /etc/app/application.properties
application.yml 新增 /etc/app/application.yml
logback.xml 修改 /etc/app/logback.xml

5.1.3 脚本文件变更

  • /opt/scripts/start.sh - 启动脚本已重构
  • /opt/scripts/start-service.sh - 新启动脚本
  • /opt/scripts/stop.sh - 停止脚本已重构
  • /opt/scripts/stop-service.sh - 新停止脚本
  • /opt/scripts/backup.sh - 备份脚本功能增强

5.2 依赖组件

5.2.1 JDK版本

从 JDK 8 升级到 JDK 11

5.2.2 数据库驱动

MySQL驱动从 5.1.47 升级到 8.0.33

5.2.3 中间件

  • Redis：5.0 → 7.0
  • Kafka：2.3 → 3.4
  • ZooKeeper：3.5 → 已移除

第6章 升级注意事项

6.1 升级前准备

  1. 备份现有数据
  2. 检查JDK 11是否已安装
  3. 确认网络连通性
  4. 准备回滚方案

6.2 升级步骤

  1. 停止旧服务
  2. 执行新版本安装脚本
  3. 配置config.yaml
  4. 启动新服务
  5. 验证功能

6.3 回滚方案

如果升级失败，请执行以下操作：

  1. 停止新服务
  2. 恢复旧版本配置
  3. 启动旧服务
  4. 验证业务恢复

6.4 已知问题

  • 首次启动后需要5分钟初始化
  • 旧版客户端兼容性问题 - 建议升级
  • 部分配置项名称发生变化，请参考迁移手册
