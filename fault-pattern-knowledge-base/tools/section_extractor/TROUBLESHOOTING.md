# 故障排除指南

## 问题：没有提取到差异项

如果工具显示"未找到任何差异项"，按以下步骤诊断：

### 1. 使用详细日志

```bash
python3 section_extractor.py your_document.docx 5.1.2.1 -v
```

查看日志中的：
- 章节范围（段落索引）
- 表格位置
- 范围检查结果

### 2. 使用诊断脚本分析文档结构

```bash
python3 debug_table_position.py your_document.docx 5.1.2.1
```

这会显示：
- 所有段落的列表和索引
- 元素序列（段落和表格的顺序）
- 表格在哪个段落之后

### 3. 检查表格是否有"类型"列

工具会查找包含以下关键词的列：
- `类型`
- `变更类型`
- `分类`
- `category`
- `type`

### 4. 检查类型列的内容

类型列中的内容必须包含以下关键词才能被分类：

**删除类型：**
- 删除
- 移除
- 废弃

**新增类型：**
- 新增
- 添加
- 引入

**变更类型：**
- 修改
- 变更
- 更新

## 示例：正确和错误的表格格式

### ✅ 正确格式

| 变更类型 | 名称 | 描述 |
|---------|------|------|
| 删除 | old_func | 旧函数 |
| 新增 | new_func | 新函数 |
| 修改 | update_func | 更新函数 |

### ❌ 错误格式（无法识别）

| 操作 | 名称 | 描述 |
|------|------|------|
| 废弃 | old_func | 旧函数 |
| 创建 | new_func | 新函数 |
| 编辑 | update_func | 更新函数 |

（列名"操作"不被识别，内容也不匹配关键词）

## 常见问题

### Q: 表格在视觉上明显在章节内，但没有被提取

**A:** 可能原因：
1. 表格在子章节中（工具只提取直接属于该章节的表格）
2. 表格和章节之间有其他特殊元素（分页符等）
3. 章节编号格式不匹配（需要"编号 标题"格式）

使用诊断脚本查看实际的文档结构。

### Q: 只提取到了部分表格

**A:** 检查：
1. 其他表格是否在子章节中
2. 使用 `-v` 查看哪些表格被跳过
3. 检查表格的"类型"列是否存在

### Q: 所有项目都被归为"变更"类型

**A:** 检查类型列的内容是否准确使用关键词。如果使用自定义词汇，可以修改 `_categorize_items` 方法。
